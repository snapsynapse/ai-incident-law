# Security Policy

## Reporting

If you discover a security issue in this repository, do not open a public issue with exploit details.

Report it privately to the maintainer first and include:

- the affected file or workflow
- reproduction steps
- impact assessment
- any suggested remediation

## Scope

This repository is a static site with local maintainer tooling. The main security-sensitive areas are:

- generated data artifacts
- URL normalization and validation logic
- GitHub Actions workflow configuration
